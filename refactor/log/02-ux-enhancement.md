# 브리핑: 사용자 경험(UX) 고도화 (Feedback System)

## 1. 개선의 필요성 (Why)2
현재 프로젝트는 에러 발생 시 브라우저 기본 `alert`을 사용하여 사용자 흐름을 중단시킵니다.
- `GEMINI.md` 원칙 중 "에러 상태는 사용자 행동 유도 메시지 포함"을 만족하기엔 `alert`은 너무 제한적입니다.
- 저장 중임을 알리는 시각적 피드백이 부족하여 사용자가 중복 클릭을 하거나 대기 상태를 오해할 수 있습니다.
- "Premium Designs" 원칙에 따라 웹 서비스의 전체적인 톤앤매너와 어울리는 피드백 시스템이 필요합니다.

## 2. 개선 방안 (How)
- **Toast 알림 시스템 도입**: 성공/에러 메시지를 화면 구석에 비침습적으로 띄워주는 Toast UI를 구현합니다 (Zustand 또는 전용 라이브러리 활용).
- **Loading State 강화**: 
    - `handleSubmit` 실행 시 버튼에 로딩 스피너 표시.
    - `disabled` 상태의 시각적 강조.
- **Inline Error Message**: 폼 유효성 검사 실패 시 필드 하단에 붉은색 헬퍼 텍스트 표시.

## 3. 기대 효과 (What)
- **부드러운 사용자 흐름**: 페이지를 이탈하거나 차단하지 않고 상태를 인지시킵니다.
- **브랜드 가치 향상**: 일관된 디자인 시스템의 피드백을 통해 전문적인 느낌을 제공합니다.
- **명확한 정보 전달**: 단순 알림을 넘어, 에러 시 어떤 행동을 해야 하는지(예: "네트워크를 확인해 주세요") 가이드할 수 있습니다.

## 4. 실행 계획 (Plan)
1.  `src/shared/feedback` 또는 `shared/modal` 내에 Toast 컴포넌트 및 Store 구현.
2.  공통 버튼 컴포넌트에 `isLoading` 프로퍼티 추가 및 UI 반영.
3.  `CourseForm` 등 주요 액션 지점에 `alert` 대신 Toast 호출 적용.
4.  폼 필드들에 에러 가시성 추가.
